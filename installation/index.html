    Vintner

    Installation¶

    In this document, we describe ways to install vintner.

    NPM¶

    vintner can be installed using npm. Ensure, that npm bin -g is in your $PATH.

    1
    npm install --global opentosca-vintner

    Yarn v1¶

    vintner can be installed using yarn (classic). Ensure, that yarn global bin is in your $PATH.

    1
    yarn global add opentosca-vintner

    Yarn v2¶

    vintner can be installed using yarn (modern). However, this will not install vintner permanently but only temporary.

    1
    yarn dlx opentosca-vintner

    Script¶

    vintner can be installed using our installation script. The script currently supports only Linux x64 and arm64. For the remaining supported platforms and architectures, see the manual installation.

    1
2
    curl -fsSL https://vintner.opentosca.org/install.sh | sudo bash -
vintner setup init

    To install a specific version, run 

    1
    curl -fsSL https://vintner.opentosca.org/install.sh | sudo VERSION=${VERSION} bash -

    Manual¶

    To manually install vintner, download the binary for your system. There are no other dependencies required. You might add the binary to your $PATH. The following example shows the installation on Linux. See below for verifying the signature of the binary.

    1
2
3
4
5
6
    wget https://github.com/opentosca/opentosca-vintner/releases/download/latest/vintner-linux-x64.xz
tar -xf vintner-linux-x64.xz
rm vintner-linux-x64.xz
mv vintner-linux-x64 /usr/bin/vintner
chmod +x /usr/bin/vintner
vintner setup init

    1

tar -xf vintner-linux-arm64.xz
rm vintner-linux-arm64.xz
mv vintner-linux-arm64 /usr/bin/vintner
chmod +x /usr/bin/vintner
vintner setup init

    1

tar -xf vintner-alpine-x64.xz
rm vintner-alpine-x64.xz
mv vintner-alpine-x64 /usr/bin/vintner
chmod +x /usr/bin/vintner
vintner setup init

    The following binaries are available:

    Platform Architecture Binary Archive Signature
    Alpine x64 link link link
    Linux arm64 link link link
    Linux x64 link link link
    Windows x64 link link link

    To check, that vintner can be executed, run

    1
    vintner --version

    To start the server, run 

    1
    vintner server start

    To uninstall all files including the binary, run the following commands. This will not undeploy currently deployed applications.

    1
2
    vintner setup clean --force
rm "$(which vintner)"

    Configuration¶

    The following environment variables can be used for configuration.

    Environment Variable Default Description
    OPENTOSCA_VINTNER_HOME_DIR ${HOME_DIR}/.opentosca_vintner

    Signature¶

    To verify a signature of a binary, first import our public key and download the respective signature. The following is a walkthrough for vintner-linux-x64 using gpg.

    First, import our public key.

    1
    curl https://vintner.opentosca.org/vintner-release.gpg | gpg --import

    Then download and verify the respective signature.

    1
2
    wget https://github.com/opentosca/opentosca-vintner/releases/download/latest/vintner-linux-x64.asc
gpg --verify vintner-linux-x64.asc
